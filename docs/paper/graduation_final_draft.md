# 基于深度学习的吸烟者行为检测系统设计与实现

## 摘要
在医院、校园、办公楼和交通站点等禁烟场景中，吸烟行为管理一直比较依赖人工巡查。这样的方式虽然直观，但在实际使用中往往存在覆盖范围有限、发现不够及时和长期维护成本较高等问题。因此，利用视频监控画面自动识别吸烟行为，具有一定的现实意义。结合本次毕业设计任务，本文围绕吸烟行为检测开展了数据整理、模型训练和展示方案设计等工作。

本文以 YOLOv8n 为基础模型，将检测目标统一为 `cigarette`、`smoking_person` 和 `smoke` 三类。针对项目中不同来源数据格式不一致、类别划分不统一的问题，完成了数据转换、标签映射、异常标签清理、数据合并和数据集划分，构建了可直接用于训练的吸烟行为检测数据集。考虑到本地实验环境以 CPU 为主，训练耗时较长，本文又在全量数据集的基础上构建了温和均衡版数据集 `smoke_bal`，用于当前阶段的实验。

在模型部分，本文先搭建了 YOLOv8n 的训练、验证、推理和导出流程，再在此基础上引入 ECA 通道注意力模块，设计 YOLOv8n+ECA 改进模型，用于对比实验。当前已经完成 baseline 模型和 ECA 模型在 `smoke_bal` 数据集上的 30 轮训练与测试验证。结果表明，baseline 模型的 Precision 为 0.526，Recall 为 0.625，mAP@0.5 为 0.520，mAP@0.5:0.95 为 0.323；ECA 模型的 Precision 为 0.513，Recall 为 0.572，mAP@0.5 为 0.494，mAP@0.5:0.95 为 0.299。整体来看，当前 baseline 模型表现更稳定，`smoke` 类效果较好，而 `cigarette` 类仍然是本课题中的主要难点。

在系统实现方面，项目已经完成训练脚本、验证脚本、预测脚本和模型导出脚本的搭建，并形成了基于 FastAPI 的网页展示方案，用于展示实验流程、检测结果和案例演示。总体来看，本文已经完成了毕业设计的主要框架，能够为论文定稿和后续答辩准备提供支撑。

**关键词**：吸烟行为检测；YOLOv8n；ECA；目标检测；FastAPI

## Abstract
Smoking behavior detection is meaningful in smoke-free areas such as campuses, hospitals, office buildings and transportation hubs. Manual inspection is often limited by coverage and efficiency. Therefore, this thesis studies an automatic smoker behavior detection method based on deep learning.

This work uses YOLOv8n as the baseline model and defines three classes: `cigarette`, `smoking_person`, and `smoke`. Multiple datasets are integrated through format conversion, label remapping, anomaly cleaning and split reorganization. Considering the CPU-only local environment, a mildly balanced dataset named `smoke_bal` is further constructed for practical experiments. Based on this dataset, the baseline model has completed 30 epochs of training. The test results reach Precision 0.526, Recall 0.625, mAP@0.5 0.520 and mAP@0.5:0.95 0.323. An ECA-enhanced YOLOv8n model is also designed for follow-up comparison.

In system implementation, the project has completed the training, validation, prediction and export scripts, and adopts a FastAPI-based web interface as the final presentation layer for thesis defense, which is more suitable for demonstration and explanation.

## 第1章 绪论

### 1.1 研究背景
吸烟不仅会危害吸烟者本人健康，也会对周围人群造成二手烟影响。在医院、教学楼、宿舍楼、办公区域以及交通枢纽等公共场所，禁烟管理一直都是比较常见的实际问题。很多场景虽然安装了监控设备，但日常管理仍然主要依靠人工巡查或人工回看监控。这样做在小范围内可以起作用，但一旦场景变大、时间拉长，就容易出现看不过来、发现不及时等情况。

从管理角度看，禁烟场景需要的并不只是“事后发现”，更重要的是尽量做到及时识别和及时处理。如果能够利用现有监控画面自动识别吸烟行为，就可以在一定程度上减轻人工负担，也便于后续扩展到告警提示、截图留存和记录统计等功能。因此，吸烟行为检测并不是一个脱离实际的研究题目，而是和公共场所管理需求联系比较紧密的应用任务。

不过，这个任务本身并不简单。首先，香烟在图像中通常属于小目标，尺寸很小，特征不明显；其次，吸烟动作会受到拍摄角度、遮挡关系和人物姿态变化的影响；再次，烟雾本身形态不固定，边界也不清晰，容易受到背景和光照条件干扰。也就是说，吸烟行为检测既有小目标检测的问题，也有复杂场景识别的问题。正因为如此，本文选择围绕该任务开展数据整理、模型训练和系统展示设计，希望形成一套较完整的毕业设计实现过程。

### 1.2 国内外研究现状
目标检测技术的发展大致经历了两个阶段。较早的一类方法主要依赖人工设计特征，再结合分类器完成目标识别，例如 Haar、HOG、SIFT 等。这些方法在简单场景下能够取得一定效果，但当目标尺度变化较大、背景复杂或者存在遮挡时，识别效果通常不够稳定。

随着深度学习的发展，基于卷积神经网络的目标检测方法逐渐成为主流。两阶段检测方法如 Faster R-CNN 在精度方面表现较好，但推理速度相对较慢；单阶段检测方法如 SSD、YOLO 系列则在实时性方面更有优势，因此更适合直接用于工程场景。近几年 YOLO 系列更新较快，从 YOLOv5、YOLOv7 到 YOLOv8，整体趋势都是在保持检测速度的同时，尽量提高精度并简化部署流程。

在吸烟行为检测这一具体方向上，已有研究大多围绕两个问题展开：一是如何提高对香烟等小目标的识别效果，二是如何让模型在复杂背景下保持较稳定的检测能力。有些工作直接使用现有目标检测模型进行训练，有些工作则尝试加入注意力机制、特征融合模块或者多尺度改进方法。从这些研究可以看出，吸烟行为检测已经有比较明确的技术路线，但在实际使用时仍然面临一些共性问题，比如数据来源比较分散、类别定义不完全统一、实验过程难以复现，或者论文里只有模型结果而缺少完整系统实现。

结合本课题现有资源，本文更关注的是把数据处理、模型训练、结果验证和展示方案串联起来，而不是单独追求某一项指标的提高。这也是本文在研究思路上与单纯模型调参实验不同的地方。

## 第2章 相关理论与关键技术

### 2.1 卷积神经网络与目标检测基本原理
卷积神经网络通过卷积核在局部感受野内共享参数，能够逐层提取图像中的边缘、纹理、形状和语义信息，是当前计算机视觉任务的重要基础。与传统手工特征方法相比，卷积神经网络能够直接从数据中学习判别特征，因此在图像分类、检测和分割等任务中表现出较强的适应能力。

目标检测是在识别目标类别的同时，进一步给出目标在图像中的位置。与单纯的图像分类相比，该任务不仅要求模型具备较强的语义表达能力，还要求模型在不同尺度、不同密度和复杂背景条件下保持较稳定的定位能力。对于本文所研究的吸烟行为检测任务而言，模型既要识别人物和烟雾，也要尽量捕捉尺寸较小的香烟目标，因此多尺度特征提取与特征融合能力具有基础性作用。

### 2.2 YOLO系列模型与YOLOv8n结构特点
YOLO 系列模型属于单阶段目标检测方法，其核心思想是在一次前向传播过程中同时完成类别预测和位置回归，从而兼顾检测速度与工程部署效率。随着版本演进，YOLO 模型在骨干网络结构、特征融合方式和检测头设计等方面不断优化，逐步形成了较成熟的轻量化检测框架。YOLO 系列不同规模模型在精度与计算开销上的差异如图2-1所示。

YOLOv8n 是 YOLOv8 系列中的轻量化版本，参数规模较小，训练和推理成本相对较低，更适合在本课题当前的本地 CPU 环境下开展实验，因此本文选择其作为基线模型。其网络结构如图2-2所示。该模型在主干网络中采用 C2f 模块提取特征，在颈部网络中进行多尺度特征融合，并通过 Anchor-Free 检测头完成目标分类与边界框回归。对于本课题而言，YOLOv8n 能够在模型复杂度、训练时长和检测性能之间取得较为平衡的结果。

### 2.3 ECA注意力机制原理
注意力机制的基本思想是在特征学习过程中突出重要信息、抑制冗余信息。ECA（Efficient Channel Attention）是一种轻量级通道注意力机制，其特点是在不显著增加参数量和计算量的前提下，实现对通道间相关性的建模。与部分需要降维再升维的通道注意力方法相比，ECA 直接通过局部一维卷积建立邻近通道之间的交互关系，结构更加紧凑，适合嵌入轻量模型中。ECA 模块结构如图2-3所示。

对于吸烟行为检测任务，香烟目标面积小、纹理细、易受背景干扰，若特征融合阶段对关键通道响应不够敏感，就可能导致漏检或定位不准。因此，本文在后续模型设计中尝试将 ECA 引入 YOLOv8n 的特征融合部分，希望在尽量控制模型复杂度的前提下，提高对细粒度目标特征的表达能力。

### 2.4 数据增强与小目标检测相关方法
数据增强是提升目标检测模型泛化能力的常用手段。通过翻转、缩放、平移、颜色扰动和多图拼接等方式，可以在不额外采集数据的情况下扩展样本分布，减轻模型对单一拍摄条件的依赖。对于监控类场景而言，光照变化、压缩噪声、拍摄角度差异和背景复杂度都会影响检测效果，因此适度的数据增强能够提升模型对实际场景的适应能力。

小目标检测则是目标检测中的典型难点。当目标在图像中占比很小时，其纹理和边界信息容易在下采样过程中丢失，导致检测头可利用的判别信息不足。针对这一问题，常见改进思路包括提高输入分辨率、加强浅层与深层特征融合、优化样本分布、引入注意力机制以及采用切片辅助推理等。本文后续在数据集构建中采用类别均衡策略，在模型设计中尝试加入 ECA 模块，均属于围绕小目标检测难点展开的具体实现。

### 2.5 Web系统相关技术基础
除了模型训练与验证之外，毕业设计还需要完成一定程度的系统化展示。Web 架构具有跨平台、易展示和便于扩展的特点，适合作为算法成果的应用承载方式。FastAPI 是基于 Python 的轻量级 Web 框架，具备接口定义清晰、与现有推理代码衔接方便、运行开销较低等特点，适合将目标检测能力封装为可调用的服务接口。

在页面组织方面，模板渲染和静态资源管理可以将系统说明、实验摘要与检测结果集中展示，便于答辩场景下的统一演示；在数据管理方面，数据库技术可用于保存模型配置、检测记录和任务状态，为结果追溯与运行管理提供支撑。对于本课题这样的原型系统，SQLite 适合作为本地轻量存储方案，而 PostgreSQL 等数据库则更适合后续扩展到多用户或长期运行场景。

此外，模型导出也是系统工程化的一部分。ONNX 作为常用的中间表示格式，有利于模型在不同推理环境之间迁移。本文当前仍以 PyTorch 权重作为主要训练与推理载体，但保留 ONNX 导出能力，有助于为后续轻量化部署和跨环境适配预留接口。

## 第3章 数据集构建与训练方案设计

### 3.1 数据来源与原始数据特点
本文所使用的数据并非来自单一标准数据集，而是由 AI Studio、Roboflow 以及后续补充的多个 YOLOv8 压缩包共同构成。多源数据带来的直接问题是类别命名不统一、标注格式不一致、图像分辨率差异较大，因此在模型训练前必须先完成数据重构与标签统一。结合 `datasets/reports` 中的准备报告，当前数据构建过程主要涉及以下几类来源：AI Studio VOC 标注数据 783 张，主要提供 `smoke` 类样本；Roboflow 数据 18269 张，主要提供 `smoking_person` 类样本；新增 `Cigarette.yolov8.zip` 数据 4663 张，主要提供 `cigarette` 类样本；新增 `smoke.yolov8 (1).zip` 数据 4898 张，主要提供 `smoke` 类样本；`smoke.yolov8.zip` 旧版压缩包 356 张，作为补充烟雾样本使用。

从原始类别分布看，多源数据之间存在明显偏斜。Roboflow 数据重映射后几乎全部对应 `smoking_person`，新增香烟数据以 `cigarette` 为主，而 AI Studio 与新增烟雾数据主要对应 `smoke`。这意味着如果直接合并训练，`smoking_person` 类实例数量会远高于另外两类，容易使模型在训练阶段偏向人物类目标，削弱对香烟本体这种关键小目标的学习效果。

原始图像尺寸也存在较大差异。根据最终数据检查报告，数据集中既包含 640×640、500×500 这类标准化样本，也包含 510×680、506×900、1536×2048、1280×720 等不同来源图像。这种异质性一方面提高了数据对多场景的覆盖能力，另一方面也增大了训练前数据标准化和标签校验的必要性。因此，本章首先解决的是“数据能否统一使用”的问题，而不是直接进入模型训练。

### 3.2 类别统一与标签映射规则
为保证训练目标明确，本文将检测任务统一定义为三类：`cigarette`、`smoking_person` 和 `smoke`。其中，`cigarette` 表示香烟本体，`smoking_person` 表示正在吸烟的人物主体，`smoke` 表示画面中的可见烟雾。统一后的三分类定义兼顾了吸烟行为识别的核心要素：人物、香烟和烟雾，既能够支撑目标检测实验，也便于后续系统展示与结果解释。

多源数据在原始标注中的类别并不一致。例如，新增香烟数据中包含 2 个原始类别，其中一个类别经映射后保留为 `cigarette`，另一类由于不属于本文任务定义而被忽略；新增旧版烟雾数据中也存在应被剔除的非目标类别；Roboflow 数据原始两类在重映射后统一并入 `smoking_person`。因此，类别统一并不是简单改名，而是“保留目标类 + 忽略无关类 + 重排类别编号”的过程。

为提高后续实验的可复现性，项目将类别映射逻辑显式写入数据准备脚本，并在中间报告中保留映射前后的类别统计信息。这样做有两个好处：一是后续如果补充新数据源，可以按同样规则接入；二是论文中的类别定义与训练配置文件中的 `names` 保持严格一致，避免出现实验结果与系统展示类别名称不一致的问题。

### 3.3 数据清洗与异常样本处理
在多源数据合并之前，本文对原始样本进行了清洗与完整性检查。数据清洗的目标主要包括：删除空标签、跳过格式错误标签行、校验框坐标是否越界、保证图像文件与标签文件一一对应，并在最终阶段再次执行全量数据检查。由于不同来源数据质量不完全一致，这一步是保证训练可靠性的关键前提。

从中间处理报告看，新增香烟数据中存在 278 个重映射后为空的标签文件和 152 条格式异常标注行；新增烟雾数据中存在 63 个空标签文件和 77 条格式异常标注行；旧版烟雾压缩包中存在 82 个空标签文件和 14 条格式异常标注行。Roboflow 数据在准备报告中还显示存在 48 个空标签文件。对于这些问题，本文在准备阶段先完成格式转换、异常行过滤和类别重映射，再在合并后的最终数据集上进行统一清理与检查。

最终清理报告显示，在合并阶段又进一步删除了 2 组空标签样本对和 6 组无效标签样本对。完成清理后，`datasets/reports/final_dataset_check_report.json` 与 `datasets/reports/balanced_dataset_check_report.json` 中均未再发现缺失标签、缺失图像、未知类别或越界框，说明最终用于训练的两套数据集都通过了完整性校验。对毕业设计而言，这种“先准备、再清理、最后复核”的处理流程比一次性手工整理更稳健，也更容易复现。

### 3.4 数据集划分与均衡策略
完成清洗和类别统一后，本文首先构建了全量融合数据集 `smoking_yolo_3cls_full`，其清洗后共有 28915 张图像，类别实例总数为：`cigarette` 7021、`smoke` 7041、`smoking_person` 25579。可见，虽然 `cigarette` 与 `smoke` 两类规模接近，但 `smoking_person` 类实例数量约为另外两类的 3.6 倍，类别不平衡问题十分突出。

全量数据集的划分结果为：训练集 24237 张，验证集 2342 张，测试集 2336 张。考虑到本地实验环境以 CPU 为主，若直接基于全量数据集进行多组对比训练，将面临训练时间过长、调参迭代效率低的问题。为兼顾实验效率与类别平衡，项目进一步构建了温和均衡版数据集 `smoke_bal`，在尽可能保留 `cigarette` 和 `smoke` 样本的前提下，对 `smoking_person` 类进行有控制的下采样。

平衡策略的核心思想是：保留所有包含 `cigarette` 或 `smoke` 的优先图像，再补充仅包含 `smoking_person` 的图像，使 `smoking_person` 的目标数量约控制在另外两类最大值的 1.5 倍。根据平衡数据集构建报告，最终 `smoking_person` 的目标数由 25579 降至 10562，而 `cigarette` 和 `smoke` 分别保持在 7021 和 7041。由此，`smoke_bal` 数据集总图像数降为 18236 张，显著降低了训练成本，同时保留了与课题核心相关的香烟和烟雾样本规模。

平衡后的数据集划分为：训练集 14416 张、验证集 1895 张、测试集 1925 张。检查报告显示，该数据集在三个划分中均实现了图像与标签一一对应，且三类目标均有分布。对本文而言，`smoke_bal` 的意义并不在于替代全量数据集，而在于为 baseline 与改进模型对比提供一个更适合当前计算资源的实验平台。

### 3.5 Baseline训练方案设计
在训练方案上，本文选用 Ultralytics 提供的 YOLOv8n 作为 baseline 模型，并通过 `scripts/train.py`、`scripts/val.py`、`scripts/predict.py` 和 `scripts/export_onnx.py` 构建完整的训练、验证、推理和导出链路。之所以选择 YOLOv8n，而不是更大规模的 YOLOv8s 或 YOLOv8m，一方面是因为当前实验环境长期依赖 CPU，轻量模型更有利于完成多组对比实验；另一方面是因为本课题还需要将模型推理能力接入网页展示系统，因此部署成本同样需要考虑。

根据训练配置文件，本文 baseline 方案的主要设置包括：输入尺寸 416、批量大小 8、优化器 AdamW、初始学习率 0.001、余弦退火学习率策略、早停耐心值 20，以及 `close_mosaic=10` 的后期增强收尾策略。这些配置兼顾了当前硬件条件下的可训练性与结果稳定性，能够支持毕业设计阶段的重复训练和对比分析。

在数据增强方面，baseline 主要沿用 Ultralytics 默认增强策略，包括 HSV 颜色扰动、水平翻转、尺度缩放、平移和前期 Mosaic。需要强调的是，这一阶段的增强设计应被理解为“基础泛化增强”，其目标是提高模型对一般光照变化、位置变化和姿态变化的适应能力，而不是直接解决复杂监控场景中的专项抗干扰问题。因此，本文在本章中将数据增强作为 baseline 训练方案的一部分，而不单独夸大为完整的抗干扰训练框架。

从工程实现角度看，`scripts/yolo_utils.py` 对路径检查、配置读取、模型构建和结果摘要输出做了统一封装，使得训练脚本保持了较清晰的入口结构。这种实现方式不仅有助于后续切换不同配置文件，也为第四章中的实验复现与结果对比提供了统一的执行基础。

### 3.6 YOLOv8n+ECA改进模型设计
为验证轻量注意力机制对吸烟行为检测任务的影响，本文在 YOLOv8n baseline 基础上设计了 YOLOv8n+ECA 改进模型。改进的核心思路不是对骨干网络进行大规模重构，而是在颈部特征融合阶段引入 ECA 模块，以增强关键通道特征的表达能力，同时尽量保持模型轻量化特性。

从代码实现来看，ECA 模块定义位于 `models/modules/eca.py`，改进网络结构位于 `models/yolov8n_eca.yaml`，并通过 `scripts/yolo_utils.py` 中的注册逻辑接入 Ultralytics 框架。模型结构摘要显示，ECA 被插入到两处特征拼接后的关键融合位置，用以在多尺度特征交互阶段强化有效通道响应。这样的设计与本课题的任务特点一致：香烟目标面积小、纹理弱，如果在特征融合阶段无法保留细粒度信息，就容易在后续检测头中发生漏检。

需要指出的是，本文设置 ECA 改进模型的主要目的在于验证“轻量注意力机制是否能改善当前任务表现”，而不是追求复杂网络堆叠。因此，改进实验强调的是结构对照关系清晰、训练链路可复现、结果便于分析。是否取得性能提升，将在第四章通过与 baseline 的对比结果作出客观说明。

### 3.7 本章小结
本章围绕吸烟行为检测任务的数据基础与训练方案进行了系统说明。首先，本文完成了多源数据的来源梳理、类别统一、异常清洗和完整性校验，构建了全量融合数据集 `smoking_yolo_3cls_full`；其次，为适应当前计算资源并缓解类别不平衡问题，进一步构建了温和均衡版数据集 `smoke_bal`；最后，在此基础上设计了 YOLOv8n baseline 训练方案和 YOLOv8n+ECA 改进模型。上述工作为后续实验对比和结果分析提供了统一、可复现的数据与模型基础。

## 第4章 实验设计与结果分析

### 4.1 实验环境与参数设置
本文实验在 Windows 环境下完成，主要软件环境包括 Python 3.13.7、PyTorch 2.11.0+cpu 和 Ultralytics 8.4.37。当前阶段未使用独立 GPU，因此模型训练主要依赖 CPU，这也是本文优先构建 `smoke_bal` 数据集而非直接在全量数据集上反复训练的重要原因。尽管训练速度受限，但训练、验证、预测和网页演示链路均已能够稳定运行，满足毕业设计阶段的实验与展示需求。

在训练参数上，baseline 与 ECA 改进实验尽量保持一致，以确保对比的公平性。两组实验均在 `smoke_bal` 数据集上训练 30 轮，输入尺寸为 416，批量大小为 8，优化器采用 AdamW，并保留余弦学习率调度和常规基础增强策略。这样设置的核心目的是减少除模型结构之外的干扰因素，使第四章的性能差异主要反映网络结构本身与任务适配性的差别。

### 4.2 评价指标
为较全面评估检测性能，本文采用 Precision、Recall、mAP@0.5 和 mAP@0.5:0.95 作为主要评价指标。其中，Precision 表示模型预测为正的结果中真正例所占比例，Recall 表示真实目标中被成功检出的比例，mAP@0.5 和 mAP@0.5:0.95 用于综合评价目标检测模型在不同 IoU 阈值下的定位与分类能力。

对于本课题而言，整体指标固然重要，但 `cigarette` 类的分项表现更具有分析价值。原因在于香烟本体是吸烟行为识别中最具挑战性的目标，其尺寸小、纹理弱、受遮挡影响大。如果整体 mAP 提高但 `cigarette` 类召回率和精度没有改善，那么模型对课题关键难点的支撑仍然有限。因此，本文在整体指标之外，特别关注三类目标的分类别 Precision、Recall 和 mAP 指标。

### 4.3 Baseline与ECA整体实验结果
本文分别在 `smoke_bal` 数据集上完成了 YOLOv8n baseline 和 YOLOv8n+ECA 的 30 轮训练，并统一在测试集上进行验证。测试结果显示，baseline 模型的 Precision 为 0.5259，Recall 为 0.6252，mAP@0.5 为 0.5201，mAP@0.5:0.95 为 0.3225；ECA 改进模型的 Precision 为 0.5114，Recall 为 0.5968，mAP@0.5 为 0.4893，mAP@0.5:0.95 为 0.2943。

从整体指标对比可见，当前实验条件下 baseline 在四项核心指标上均优于 ECA 模型，说明轻量注意力模块的加入并未在本次实验中带来预期提升。考虑到两组实验使用了相同的数据集、相同轮次和基本一致的训练参数，这一差异具有直接的对照意义。

表4-1给出了两组模型在测试集上的整体指标对比。

| 模型 | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 |
| --- | --- | --- | --- | --- |
| YOLOv8n baseline | 0.526 | 0.625 | 0.520 | 0.323 |
| YOLOv8n+ECA | 0.511 | 0.597 | 0.489 | 0.294 |

若从训练末期的摘要结果进一步比较，两组模型的损失收敛情况也存在差异。`runs/reports/baseline_vs_eca_balanced_v2.json` 显示，baseline 的最终 box loss、cls loss 和 dfl loss 分别为 1.3987、1.3724 和 1.3136，均低于 ECA 模型对应的 1.4891、1.4661 和 1.5515。这说明在相同训练预算下，baseline 的优化过程更加稳定，而 ECA 模型并未形成更优的收敛趋势。

### 4.4 分类别结果对比分析
为了进一步分析两组模型的差异，本文对 `cigarette`、`smoking_person` 和 `smoke` 三类目标分别统计了 Precision、Recall、mAP@0.5 和 mAP@0.5:0.95。结果显示，baseline 模型在三类目标上的表现更均衡，其中 `smoke` 类效果最佳，`cigarette` 类依然是最薄弱环节。ECA 模型虽然在 `smoke` 类上保持了较高水平，但在 `cigarette` 和 `smoking_person` 两类上均未超过 baseline。

特别是与课题核心难点直接相关的 `cigarette` 类，baseline 的 Recall 为 0.6403，mAP@0.5 为 0.3207，而 ECA 模型分别为 0.6199 和 0.2702。也就是说，在最需要关注的小目标类别上，ECA 并未改善漏检问题，反而在综合定位指标上出现较明显下降。这一结果意味着，当前改进模块并没有有效强化香烟类目标的判别特征。

### 4.5 训练过程与收敛情况分析
从训练过程看，baseline 与 ECA 两组实验在前期均出现了较明显的性能上升，但随着轮次增加，提升幅度逐步减小。结合 `results.csv` 与训练摘要结果可以看出，baseline 在训练后期保持了更低的损失水平，说明其对当前数据集的适配更稳定；ECA 模型虽然训练链路完整打通，但在 30 轮预算内并未表现出更优的收敛趋势。

baseline 模型的训练结果曲线如图4-2所示。综合精度、召回率和 mAP 指标的变化可以看出，模型在前期收敛较快，后期进入相对平稳阶段，这与当前实验所采用的轻量网络规模和 30 轮训练预算基本一致。

这一现象从工程角度是可以理解的。首先，当前实验使用的仍是轻量模型与中等输入尺寸配置，ECA 模块的收益可能需要更长训练周期或更高分辨率输入才能体现；其次，`smoke_bal` 数据集虽然缓解了类别不平衡，但香烟小目标的判别难度并未根本改变，因此仅通过通道注意力增强未必足以带来显著收益；再次，在 CPU 训练条件下，实验轮次和调参空间受到限制，也可能削弱结构改进所带来的潜在提升。

因此，本章更合理的结论不是“ECA 无效”，而是“在当前数据规模、训练预算和部署约束下，ECA 并未表现出优于 baseline 的实际收益”。这样的结论更符合实验事实，也更适合作为后续优化方向选择的依据。

### 4.6 误检与漏检案例分析
为了使结果分析更加具体，本文进一步结合 `cigarette_analysis.json` 对香烟类检测中的误检与漏检现象进行了专项分析。该报告针对优先复核样本统计显示，在抽取的 20 张含香烟图像中，共出现 22 个漏检香烟标注框，而高置信度误检数量为 0，低置信度匹配样本为 3。由此可以看出，当前 baseline 模型在香烟类上的主要问题并不是“把错误目标大量识别成香烟”，而是“在难样本上未能稳定检出香烟”。

这种现象与课题任务特点高度一致。香烟目标在图像中通常面积较小，且经常与手指、嘴部区域、高亮边缘或细长背景结构同时出现。当目标发生遮挡、模糊或与背景对比度较低时，模型更容易出现漏检而不是高置信度误检。换言之，当前主要矛盾是召回能力不足，而不是置信度阈值设置过宽。

对于 `smoking_person` 类，问题主要体现在复杂姿态和多人遮挡场景下的稳定性。当人物仅出现局部区域、吸烟动作不明显或多个人物交叉重叠时，模型输出会出现波动。`smoke` 类虽然整体效果较好，但在逆光、弱光和背景色接近的情况下，仍然会出现边界模糊导致的漏检。这说明当前系统虽然对典型场景具备一定识别能力，但距离复杂监控环境下的稳定应用仍有差距。

### 4.7 结果讨论与原因分析
综合前述实验和专项分析结果，本文认为当前 ECA 模型未能超过 baseline，主要原因可以从三个方面理解。第一，任务瓶颈首先来自数据问题而非单一结构问题。`cigarette` 类作为典型小目标，其判别难点在于目标尺度、遮挡和标注质量，而不是单纯的特征通道表达不足。第二，当前实验轮次和输入尺寸受 CPU 环境限制，难以为改进模块提供更充分的训练预算。第三，ECA 插入于颈部特征融合位置后虽然增强了通道建模能力，但对于极小目标的几何定位与尺度表达提升有限，因此未能直接转化为香烟类 mAP 的提升。

从论文写作角度看，这一结果具有正面意义。它说明本文并不是通过人为筛选结果来证明“改进一定有效”，而是保留了真实实验中常见的情况：结构修改可运行，但不一定优于基线模型。这样的结论更客观，也更容易支撑后续对优化方向的重新判断。

同时，baseline 结果优于 ECA 并不意味着本文的改进尝试缺乏价值。相反，正是由于改进模型的训练链路、验证链路和对比分析过程均已完成，本文才能较明确地指出当前任务的主要限制因素仍然集中在香烟小目标样本与复杂场景泛化能力，而不是简单归因于“模型不够复杂”。

### 4.8 场景适用性与技术边界分析
从当前实验结果和系统演示情况来看，本文提出的方案已经能够覆盖一部分典型禁烟监控场景，例如室内办公区、教学楼走廊、实验室门口等固定摄像头视角下的图片检测任务。在这些场景中，如果人物较明显、画面清晰度尚可且光照变化不过于剧烈，baseline 模型通常能够给出较稳定的检测结果，这也是其被选为当前主模型的重要原因。

但如果进一步追问该方案是否足以支撑所有场景识别，答案是否定的。对于夜间弱光、逆光背影、多人交叉遮挡、远距离小目标、室外复杂背景、低分辨率压缩视频以及烟雾边界极不明显的场景，当前模型仍然存在明显波动。尤其是 `cigarette` 类目标，由于尺寸小、纹理弱、容易与高亮细长区域混淆，因此很难在所有复杂场景中保持稳定识别。

因此，本文更适合将当前成果界定为“面向典型场景的吸烟行为检测系统原型”。该原型已经验证了多源数据整合、轻量模型训练和网页展示的可行性，但并不应被表述为“已经实现全场景吸烟行为识别”。这种边界说明既符合实验事实，也能使论文结论更稳妥。

### 4.9 本章小结
本章围绕实验设计与结果分析，对 YOLOv8n baseline 和 YOLOv8n+ECA 两组模型在 `smoke_bal` 数据集上的表现进行了系统比较。实验结果表明，baseline 在整体指标和分类别指标上均优于 ECA 改进模型，当前最主要的性能瓶颈仍集中在 `cigarette` 类小目标检测。通过对目标尺寸分布、误检漏检现象以及训练收敛情况的综合分析，可以认为本文已经得到一组真实、可解释且能够支撑后续系统实现的实验结论。

## 第5章 系统实现与展示设计

### 5.1 系统总体架构
本课题的系统实现围绕“数据准备—模型训练—网页展示”三条主线展开，形成了职责清晰、层次明确的工程结构。数据准备层负责原始压缩包整理、类别映射、异常标签清理、训练集划分与完整性检查；模型训练层负责 YOLOv8n baseline、YOLOv8n+ECA 及相关对比实验的训练、验证、推理和导出；应用展示层则负责将训练得到的权重、实验摘要与检测结果组织为可直接演示的 Web 系统。这样的分层方式既便于分别调试，也便于在论文中对数据、算法和应用三个层面的工作进行对应说明。

从当前项目目录来看，`datasets/raw`、`datasets/interim` 与 `datasets/final` 构成数据集加工链路，`scripts` 目录承担数据处理、训练、验证、预测、导出、实验汇总与系统检查等入口脚本，`models` 目录保存改进模型结构定义，`app` 目录负责网页服务、推理封装与数据库管理。训练产物统一保存在 `runs/train`、`runs/val` 和 `runs/reports`，系统运行过程中产生的上传文件、结果文件和数据库文件统一保存在 `output/web_demo`。这种目录划分保证了实验结果、系统输出与原始数据之间的边界较为明确，能够减少后期整理论文与复现实验时的混乱。

系统运行时，前端页面通过浏览器发起请求，FastAPI 服务负责路由分发、输入校验与结果组织，`DetectionService` 负责模型加载、图像推理和视频处理，数据库与文件系统共同负责保存结构化记录和大体积结果文件。整个数据流从“用户上传—接口接收—模型推理—结果回写—页面展示”逐步展开，形成较完整的业务闭环。系统总体架构如图5-2所示。

### 5.2 推理与导出流程实现
为保证实验结果能够从“训练完成”顺利过渡到“可演示、可复核、可部署”，本项目在训练脚本之外，进一步实现了验证、预测、导出和实验汇总等配套流程。`scripts/val.py` 用于对指定权重开展验证并生成测试摘要，`scripts/predict.py` 用于对图片或视频执行离线推理，`scripts/export_onnx.py` 用于将训练得到的权重导出为 ONNX 模型，`scripts/summarize_experiments.py` 与 `scripts/summarize_cigarette_experiments.py` 用于汇总不同实验的关键指标。通过这些脚本，可以将训练结果转化为后续章节所需的对比表、案例图和系统演示素材。

在推理流程上，系统默认以训练阶段得到的 `best.pt` 作为主权重文件，先通过验证脚本确认主要指标，再由预测脚本在样例图片、视频文件或网页上传文件上执行检测。对毕业设计而言，这一流程的价值在于：论文中的实验结论不再停留于训练日志，而是能够通过独立的验证与推理步骤重新得到，增强了结果的可复查性。

在导出流程上，ONNX 主要承担后续部署接口的角色。当前网页系统仍以 PyTorch 权重作为实际推理后端，但保留 ONNX 导出能力，可以为后续部署到更轻量的推理环境或与其他前端系统对接提供基础。换言之，ONNX 在本课题中并非展示层的核心依赖，而是系统工程化的预留出口。

### 5.3 网页展示方案设计
在毕业设计答辩场景下，系统展示不仅需要给出检测结果，还需要同时呈现模型信息、实验摘要、检测记录和运行状态。因此，展示层不能仅停留在单次推理结果回显，而需要具备一定的页面组织能力和流程管理能力。基于这一需求，本文最终采用浏览器访问的网页展示方案，将模型能力、实验信息与结果管理统一到同一套界面中进行呈现。

相较于单机脚本式演示，网页方案具有三方面优势。其一，展示入口统一，评审人员无需依赖本地开发环境即可直观看到系统主界面、检测结果和历史记录；其二，前后端职责分离，界面展示与推理逻辑解耦，便于后续更换模型权重或调整页面布局；其三，网页结构更适合答辩投屏，能够在同一界面下集中展示课题背景、模型配置、图片检测和视频任务等内容，从而增强系统部分的完整性表达。

基于上述考虑，本文将网页展示方案定位为算法实验的应用承载层，而不是独立于模型之外的附加界面。也就是说，网页系统并非替代训练和验证流程，而是对前文实验结果的进一步组织与封装，使课题成果从离线指标分析延伸到可访问、可交互、可追踪的应用原型。

### 5.4 FastAPI 前端展示设计
当前系统的网页端以后端驱动为主，核心入口位于 `app/web_demo.py`。该文件基于 FastAPI 构建服务框架，通过 `Jinja2Templates` 返回页面模板，并通过 `StaticFiles` 暴露静态资源与检测结果文件。首页负责展示系统概览，`/api/health` 用于健康检查，`/api/dashboard` 用于返回数据集规模、实验摘要与系统状态，`/api/models` 与 `/api/settings` 用于模型管理和参数配置，`/api/detect/image` 与 `/api/tasks/video` 分别负责图片检测和视频任务调度。由此，页面展示、接口调用与推理逻辑被清晰拆分，便于定位问题和后续扩展。

前端展示并未追求复杂交互，而是围绕毕业设计所需的核心信息展开组织：其一是展示数据集规模、实验指标和模型状态，使读者能够快速理解模型训练基础；其二是提供图片检测入口，直观展示目标框、类别和置信度；其三是提供检测记录与视频任务信息，使系统不再局限于单次图片预测，而具备基础的运行管理能力。就答辩场景而言，这种“管理控制台式”的前端结构比单一检测页更能体现项目的完整性。

网页端检测结果示例如图5-1所示。该界面能够直观呈现输入图像中的检测框、类别标签和置信度信息，同时保留结果文件路径与记录信息，便于在答辩过程中说明系统从上传、推理到结果回显的完整流程。

从一次图片检测的执行链路看，用户在页面上传图片后，请求首先进入 FastAPI 接口层；接口层完成文件类型和大小校验后，将二进制数据交由 `DetectionService` 解码与推理；推理结果再被整理为检测框列表、标注图像路径和记录编号，同时写入数据库与结果目录；最后前端页面根据接口返回内容进行结果展示。视频任务的处理逻辑与此类似，只是增加了后台任务、进度回写和结果视频文件管理步骤。由此，网页系统已经具备从输入、推理到结果回显与记录保存的完整闭环。

### 5.5 核心代码模块实现
从代码结构看，系统的第一类核心模块是训练与数据工具模块。`scripts/train.py` 负责读取配置并发起训练，`scripts/yolo_utils.py` 负责模型构建、路径检查与通用配置处理，`scripts/dataset_utils.py`、`scripts/check_dataset.py`、`scripts/build_balanced_dataset.py` 等脚本共同承担数据清理、重构与检查任务。这部分代码保证了数据集加工和实验执行过程具有一致的入口与输出约束，是整套系统能够稳定复现的基础。

第二类核心模块是模型结构与实验分析模块。`models/yolov8n_eca.yaml` 和 `models/modules/eca.py` 给出了 ECA 模块接入 YOLOv8n 的实现方式，`scripts/analyze_cigarette_detection.py` 和 `scripts/summarize_cigarette_experiments.py` 则围绕 `cigarette` 类的小目标检测效果开展专项分析。它们的意义不仅在于完成模型改进尝试，还在于为论文第四章和第六章提供了更有针对性的实验依据，使“香烟类识别仍是难点”这一结论具有可追溯的分析来源。

第三类核心模块是应用服务模块。`app/web_demo.py` 负责路由组织、请求分发和接口返回；`app/utils/web_inference.py` 中的 `DetectionService` 封装了权重选择、上传校验、图像解码、模型推理、标注回写和视频处理等关键逻辑；`app/config.py` 负责运行时目录与数据库地址配置；`app/db.py` 和 `app/db_models.py` 负责数据库初始化、会话管理和表结构定义。通过这些模块的协作，模型能力被转换为可直接访问的网页服务，这也是论文从算法实验走向系统实现的关键一步。

### 5.6 数据库与历史记录模块设计
与早期仅用于演示单次预测结果的原型不同，当前网页系统已经引入了数据库持久化机制。系统默认采用 SQLite，将数据库文件保存在 `output/web_demo/smoker_behavior.db`；当设置环境变量 `SMOKER_DB_URL` 后，也可以切换到 PostgreSQL 等后端。`app/db.py` 在服务启动阶段自动完成表创建和默认配置初始化，使数据库成为系统的正式组成部分，而非仅停留在概念层面的扩展设想。

数据库模块的设计目标主要有三点：一是保存模型注册信息和默认运行参数，避免每次启动都依赖手工指定权重；二是保存图片检测记录及其框级结果，便于后续查询、复核和论文案例整理；三是保存视频任务状态、处理进度和输出路径，使较长时间的视频处理任务具备可跟踪性。结合当前项目实现，数据库已经承担起系统配置管理与结果持久化的双重作用。

从关系结构上看，`model_registry` 负责保存可用模型及其权重路径，`app_settings` 通过 `default_model_id` 指向默认模型配置，`image_detections` 与 `video_tasks` 则分别保存图片检测和视频任务的主记录；其中，`image_detection_boxes` 通过外键与 `image_detections` 形成一对多关系，用于保存单次图片检测中的框级明细结果。数据库关系说明如图5-3所示。该结构既能满足答辩演示阶段的轻量化需求，又为后续扩展模型管理、结果检索与任务追踪提供了清晰的数据基础。

表5-1给出了当前系统中主要数据表及其作用说明。

| 数据表 | 关键字段 | 作用说明 |
| --- | --- | --- |
| `model_registry` | `name`、`weights_path`、`device`、`is_available` | 记录可切换模型及其权重路径，支持默认模型管理 |

### 5.7 系统稳定性设计
本课题的系统稳定性设计主要围绕“训练可复现、服务可启动、输入可校验、异常可追踪”四个方面展开。在训练与验证链路中，相关脚本会先检查数据集路径、配置文件和权重文件是否存在，再执行实际任务；在展示链路中，系统启动时会优先初始化数据库、注册默认模型并载入运行参数，从而减少因环境配置不完整导致的启动失败。

在输入处理层面，`DetectionService` 对图片和视频上传均设置了类型、后缀和体积限制，并在图像解码失败、文件为空、文件过大或格式不支持时返回明确错误信息。对于视频任务，系统采用后台队列方式异步处理，并通过进度回调持续更新已处理帧数、检测数量和任务状态。这样即使视频处理耗时较长，前端也能够通过接口查询任务进度，而不会因为同步等待而导致页面阻塞。

在运行管理层面，系统默认优先加载已验证过的 baseline 权重，同时保留模型切换接口，使演示过程能够在稳定性和灵活性之间取得平衡。`/api/health` 接口用于检查权重文件是否存在、数据库是否就绪以及当前推理配置是否可用，能够在答辩前快速确认系统状态。对于异常任务，数据库中的状态字段与错误信息字段也能够保留运行轨迹，为后续排查提供依据。除此之外，系统还在视频处理链路中加入了时间平滑相关参数，用于降低帧间检测抖动对结果呈现造成的影响，从而提升演示视频中的输出稳定性。

此外，系统输出目录、检测记录和任务结果均采用统一的命名与存储规则，减少了临时文件散落和路径混乱的问题。对毕业设计而言，这些稳定性设计未必构成复杂的软件工程机制，但它们直接提高了系统在实际演示中的可控性，是论文系统部分不可忽视的实现内容。

### 5.8 系统测试
结合本课题目前的规模与答辩需求，系统测试采用“人工核验 + 自动冒烟检查”相结合的方式开展。人工测试主要用于确认页面交互、模型切换和结果展示是否符合预期；自动测试则通过 `scripts/system_smoke_test.py` 调用 FastAPI `TestClient`，对首页、健康检查、仪表盘、图片检测、记录查询和视频任务等关键链路进行集中验证。该脚本会自动构造样例图片与样例视频，并将检测结果写入 `runs/reports/system_smoke_test.json`，从而形成可复用的测试报告。

## 第6章 总结与展望

### 6.1 全文总结
本文围绕“基于深度学习的吸烟者行为检测系统设计与实现”这一主题，完成了数据集整理、模型训练、实验分析和系统实现四个层面的工作。首先，针对多来源数据格式不一致、类别定义不统一的问题，本文完成了 VOC 到 YOLO 的转换、标签重映射、异常标签清理与训练集划分，构建了用于三分类检测任务的吸烟行为数据集，并进一步生成了适合 CPU 训练环境的平衡版数据集 `smoke_bal`。这一过程为后续模型训练提供了可复现的数据基础。

在模型设计与实验部分，本文以 YOLOv8n 为 baseline，构建了标准训练、验证与推理流程，并在此基础上引入 ECA 注意力模块开展改进实验。实验结果表明，在当前数据规模、输入分辨率与 CPU 训练条件下，YOLOv8n baseline 的整体表现优于加入 ECA 的改进模型，说明轻量注意力机制并不必然带来与本任务相匹配的性能收益。与此同时，围绕 `cigarette` 类开展的专项分析进一步表明，小目标、遮挡和复杂背景仍是影响系统性能的主要因素。

在系统实现部分，本文并未停留于离线训练结果展示，而是进一步完成了以 FastAPI 为核心的网页管理与演示系统。该系统具备模型注册与切换、默认参数配置、图片检测、视频任务处理、记录查询与结果持久化等功能，并通过 SQLite/PostgreSQL 兼容的数据库设计实现了基础运行管理。由此，论文中的算法实验被落实为可实际访问、可演示、可保存结果的应用原型。

综合来看，本文完成的工作重点不在于追求单一指标的极限提升，而在于建立一条较完整的课题实现链路：数据可整理，模型可训练，结果可分析，系统可展示。对于毕业设计而言，这种从算法验证到系统落地的贯通式实现，更能体现课题的完整性与工程价值。

### 6.2 存在不足
尽管本文已经完成了课题的主体工作，但当前研究仍存在若干不足。首先，`cigarette` 类目标在图像中通常尺寸较小，且经常伴随遮挡、模糊、逆光和背景干扰，因此仍是漏检最集中的类别。现有数据虽然覆盖了人物、烟雾和香烟三类目标，但针对复杂监控场景的难样本积累仍不充分，这在一定程度上限制了模型在真实环境中的稳定性。

其次，本文主要训练工作是在本地 CPU 环境下完成的，训练轮次、输入尺寸和对比实验数量均受到较大约束。虽然已经完成 baseline、ECA 及相关验证流程，但尚未在更充足的 GPU 条件下开展更大规模的消融实验和更长周期训练，因此当前结论更适合界定为面向毕业设计阶段的阶段性结果。

最后，系统部分已经实现了网页展示、检测记录保存和视频任务管理等基础功能，但整体仍属于原型系统。对于长期运行场景而言，实时视频流处理能力、异常恢复机制、权限管理和大规模数据存储能力仍需进一步完善。因此，现阶段系统更适合作为算法验证与答辩展示平台，而非直接投入复杂场景的长期部署。

### 6.3 后续展望
后续工作首先应继续围绕复杂场景下的抗干扰训练展开。可以结合现有误检与漏检样本，进一步补充遮挡、模糊、压缩噪声和逆光场景数据，并通过难样本重采样、针对性增强和持续清洗等方式提升模型对真实监控环境的适应能力。这样既能够改善香烟类目标的召回表现，也有助于提高整体检测稳定性。

其次，在模型优化方面，后续研究应更加聚焦于小目标检测性能提升与轻量部署之间的平衡。除继续调整输入分辨率和特征融合结构外，还可以尝试知识蒸馏、多尺度训练和切片辅助检测等策略，以提高香烟等小目标在复杂背景中的识别能力。这些方法是否有效，仍需结合更充分的 GPU 实验资源进行验证。

最后，在系统工程方面，后续可在现有网页原型基础上继续完善平台化能力，包括优化前端展示效果、增强视频任务管理、补充结果检索与导出功能，并逐步提升系统的部署灵活性与运行稳定性。若后续能够结合更完整的实验分析和更长期的运行测试，系统将更接近实际应用场景的需求。

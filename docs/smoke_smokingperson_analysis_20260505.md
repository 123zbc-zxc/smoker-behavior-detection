# Smoke 和 Smoking_Person 检测效果不佳原因分析

**分析日期：** 2026-05-05  
**问题：** 为什么在实际图片/视频检测中，smoke（烟雾）和smoking_person（吸烟者）识别效果不理想？

---

## 一、性能数据回顾

### 测试集指标对比（hard_finetune模型）

| 类别 | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 | 置信度阈值 |
|------|-----------|--------|----------|--------------|-----------|
| cigarette | 0.4456 | **0.7472** | 0.4967 | 0.2989 | 0.12 |
| **smoking_person** | **0.4331** | **0.5969** | **0.3962** | **0.2494** | 0.22 |
| smoke | **0.7429** | 0.7258 | **0.7883** | 0.5195 | **0.28** |

### 表面矛盾

- **smoke指标看起来很好**（mAP@0.5=0.7883，最高），但实际使用效果不佳
- **smoking_person指标最差**（mAP@0.5=0.3962），实际效果也确实差

---

## 二、Smoking_Person 检测效果差的原因

### 原因1：数据严重不足（主要原因）

#### 数据集统计

从 `balanced_dataset_report.json` 分析：

| 阶段 | cigarette | smoking_person | smoke |
|------|-----------|----------------|-------|
| 原始数据 | 7,021 | **7,041** | 25,579 |
| 均衡后训练集 | 7,021 | **7,021** | 10,562 |
| 测试集 | 1,077 | **784** | 1,043 |

**问题分析：**
1. smoking_person原始样本就最少（7,041），无法像smoke那样通过下采样获得更多样性
2. 测试集中smoking_person只有784个实例，比其他两类少25-33%
3. 均衡策略只是简单截断，没有通过数据增强补充smoking_person样本

**结果：** 模型在训练时见到的smoking_person样本最少，学习最不充分。

---

### 原因2：任务本质难度高

#### 人体检测的复杂性

**Smoking_person不是简单的"人"检测，而是"正在吸烟的人"检测：**

1. **姿态多样性**
   - 站立、坐着、躺着、侧身、背对
   - 手臂遮挡面部
   - 只露出上半身或局部

2. **标注歧义**
   - 什么算"吸烟者"？手持香烟但未吸？刚吸完？
   - 框应该框全身还是上半身？
   - 多人场景中如何区分吸烟者和旁观者？

3. **与cigarette的关联性**
   - 理论上smoking_person应该与cigarette共现
   - 但实际标注中可能不一致（有人框了但没框烟，或反之）
   - 模型学到的是"人"的特征还是"吸烟动作"的特征？

#### 证据：Hardcase标注工作区

项目中有 `datasets/hardcase_labeling_workspace/` 目录，其中：
- 120张困难样本图片
- **70张是空标签**（没有任何标注）
- 说明存在大量"难以判断是否吸烟"的边界案例

**推测：** 这些空标签图很可能是smoking_person类别的歧义样本。

---

### 原因3：模型容量分配不均

#### YOLOv8n的资源分配

YOLOv8n只有3.2M参数，要同时学习3个差异巨大的类别：
- cigarette: 极小目标（<20px²）
- smoking_person: 大目标（可能占图像30%）
- smoke: 不定形状

**训练过程中的梯度竞争：**
- cigarette因为小目标难度高，占用了大量模型容量
- smoke因为样本多（10,562 vs 7,021），梯度信号强
- smoking_person夹在中间，既没有"小目标"的特殊关注，也没有"大样本量"的优势

**结果：** smoking_person成为"被牺牲"的类别。

---

### 原因4：置信度阈值设置

当前阈值：
- cigarette: 0.12（极低，保召回）
- smoking_person: 0.22（中等）
- smoke: 0.28（高，保精度）

**问题：** smoking_person的0.22阈值是"妥协值"，既不够低（无法提升召回），也不够高（无法过滤误检）。

从测试集结果看：
- Recall=0.5969 → 40%的smoking_person被漏检
- Precision=0.4331 → 56%的检测是误检

**实际效果：** 用户看到的是"该检出的没检出，不该检出的一堆误报"。

---

## 三、Smoke 检测效果差的原因（指标好但实际差）

### 矛盾现象

- **测试集指标优秀**：mAP@0.5=0.7883，Precision=0.7429
- **实际使用感觉差**：用户反馈效果不好

### 原因1：高置信度阈值牺牲了召回

**阈值设计逻辑：**

从 `video_threshold_temporal_search.json` 分析，当前阈值是为**视频时序稳定性**优化的，不是为静态图片准确率优化的。

```
优化目标：85%视频事件命中率（20个测试视频）
结果：smoke阈值=0.28（最高）
```

**为什么smoke阈值最高？**
- Smoke容易产生大量低置信度检测（因为边界模糊）
- 低置信度检测会导致视频中检测框闪烁
- 提高阈值到0.28可以过滤掉这些"不稳定"的检测

**代价：** 很多真实的smoke被过滤掉了。

#### 数学分析

```
测试集 Recall = 0.7258
意味着：27.4%的smoke目标被漏检

如果降低阈值到0.20：
预期 Recall 可能提升到 0.80-0.85
但 Precision 会下降到 0.60-0.65
```

**用户感受：** "明明有烟雾，为什么检测不出来？" → 因为置信度<0.28被过滤了。

---

### 原因2：Smoke的本质特性

#### 烟雾检测的固有难题

1. **无明确边界**
   - 烟雾是渐变的，从浓到淡没有清晰分界
   - 标注时人工画的框是主观的
   - 模型学到的是"大致区域"而非精确边界

2. **透明度与背景融合**
   - 烟雾半透明，与背景混合
   - 白色背景中的白烟几乎不可见
   - 光照变化严重影响烟雾可见度

3. **形状多变**
   - 刚吐出的烟雾是细流
   - 扩散后的烟雾是大片云状
   - 风吹后的烟雾是不规则形状

4. **时间依赖性**
   - 烟雾是动态的，单帧图片可能只捕捉到部分
   - 视频中连续帧才能完整看到烟雾轨迹

**结论：** Smoke的高mAP@0.5（0.7883）已经是在这些困难条件下的"不错"表现，但对用户来说仍然"不够好"。

---

### 原因3：数据集偏差

#### 原始数据来源

项目使用了5个数据集，其中包括：
- D-Fire smoke/fire数据集（火灾烟雾）
- HMDB51 smoke视频（行为识别数据集）

**问题：**
- 火灾烟雾（浓烟）与香烟烟雾（淡烟）特征差异大
- 行为识别数据集可能标注粗糙（只要有吸烟行为就标，不精确框烟雾）

**结果：** 模型学到的smoke特征可能偏向"浓烟"，对"淡烟"效果差。

---

## 四、实际案例分析

### 案例1：视频检测中的表现

从 `video_threshold_temporal_search.json`：
- 20个测试视频，17个命中（85%）
- 平滑命中帧比例：0.2997（约30%的帧有检测）

**解读：**
- 85%视频命中率看起来不错
- 但只有30%的帧有检测 → 70%的帧是"空白"的
- 对于连续吸烟的视频，这意味着大量帧漏检

**用户感受：** "视频里明明一直在吸烟，为什么检测框时有时无？"

---

### 案例2：TTA+SAHI增强推理的效果

从最新的TTA+SAHI验证（50张测试图）：

| 模式 | smoking_person Recall | smoke Recall |
|------|----------------------|--------------|
| normal | 0.2000 | 0.6786 |
| tta+sahi | 0.2667 | 0.7143 |

**smoking_person提升：** 0.20 → 0.27（+33%相对提升）  
**smoke提升：** 0.68 → 0.71（+5%相对提升）

**结论：** TTA+SAHI对smoking_person帮助更大，说明这个类别确实存在"模型能力不足"的问题。

## 五、改进建议

### 针对 Smoking_Person

#### 短期方案（答辩前可做）

1. **降低置信度阈值**
   ```python
   # 从0.22降到0.15-0.18
   CLASS_CONF_THRESHOLDS = {
       0: 0.12,
       1: 0.15,  # 原来是0.22
       2: 0.28,
   }
   ```
   预期：Recall从0.60提升到0.70-0.75，代价是Precision下降

2. **启用TTA+SAHI推理**
   - 已验证可提升33%相对召回率
   - 适合答辩演示和离线分析

#### 长期方案（论文改进章节）

1. **数据增强专项**
   ```yaml
   # 针对smoking_person的增强
   copy_paste: 0.20        # 把人物粘贴到更多场景
   mixup: 0.15             # 混合不同姿态的人
   mosaic: 0.8             # 增加多人场景
   ```

2. **收集更多样本**
   - 目标：smoking_person样本量翻倍（从7,021到15,000+）
   - 重点：不同姿态、遮挡、光照条件

3. **两阶段检测**
   ```
   Stage 1: 通用人体检测（YOLOv8预训练）
   Stage 2: 吸烟动作分类（在检测到的人框内判断是否吸烟）
   ```

4. **类别加权Loss**
   ```python
   # 训练时给smoking_person更高权重
   class_weights = [1.0, 2.0, 0.8]  # cigarette, smoking_person, smoke
   ```

---

### 针对 Smoke

#### 短期方案

1. **双阈值策略**
   ```python
   # 静态图片用低阈值（提升召回）
   STATIC_THRESHOLDS = {2: 0.20}
   
   # 视频用高阈值（保持稳定）
   VIDEO_THRESHOLDS = {2: 0.28}
   ```

2. **后处理优化**
   ```python
   # 对smoke检测框做形态学膨胀，扩大边界
   # 合并相邻的小smoke框为大框
   ```

#### 长期方案

1. **分割模型辅助**
   - 用语义分割模型（如U-Net）精确定位烟雾边界
   - 检测模型负责"有无"，分割模型负责"边界"

2. **时序建模**
   - 对视频，用光流或3D卷积捕捉烟雾运动
   - 利用"烟雾会扩散"的先验知识

3. **多模态融合**
   - 结合RGB和红外图像（烟雾在红外下更明显）
   - 或使用深度信息辅助

## 六、答辩建议表述

### 可以说

> "项目在三类目标检测中，smoking_person类别表现相对较弱（mAP@0.5=0.396），主要原因是：（1）训练样本量不足且姿态多样性有限；（2）标注存在歧义，'吸烟者'的定义边界不清晰；（3）YOLOv8n模型容量有限，在多尺度目标学习中对该类别的资源分配不足。Smoke类别虽然测试集指标较好（mAP@0.5=0.788），但实际应用中因采用较高置信度阈值（0.28）以保证视频检测稳定性，牺牲了部分召回率。针对这些问题，建议通过数据增强、两阶段检测和类别加权Loss等方法改进。"

### 不能说

- ❌ "smoking_person和smoke检测完全失败" —— 指标显示仍有一定效果
- ❌ "这两个类别无法检测" —— 是效果不够好，不是完全不行
- ❌ "模型有bug" —— 是设计权衡和数据限制，不是实现错误

## 七、核心结论

### Smoking_Person 效果差的根本原因

1. **数据不足**（784测试样本 vs 1077 cigarette）
2. **任务难度高**（姿态多样、标注歧义）
3. **模型容量不足**（YOLOv8n无法同时学好3个差异大的类别）

### Smoke 效果差的根本原因

1. **高阈值策略**（0.28为视频稳定性优化，牺牲召回）
2. **本质特性**（无边界、透明、多变）
3. **数据集偏差**（火灾浓烟 vs 香烟淡烟）

### 最重要的启示

**指标好≠实际效果好。** Smoke的mAP@0.5=0.788在测试集上很高，但因为：
- 测试集可能偏向"明显烟雾"样本
- 置信度阈值为视频场景优化，不适合静态图片
- 用户期望与模型能力不匹配

**改进优先级：** Smoking_person > Smoke，因为前者是根本性的数据和模型问题，后者更多是阈值和后处理问题。

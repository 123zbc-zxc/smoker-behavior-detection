# 论文与答辩可用结论（2026-05-03）

## 模型选择

本文最终选用 YOLOv8n 640px Hard Fine-tune (20260502) 作为系统默认模型。与旧冠军模型相比，该模型在测试集上的整体 Recall 略有提升，cigarette 类别 Recall 从 0.7212 提升到 0.7472，更符合吸烟行为检测中“小目标尽量发现”的任务需求。需要客观说明的是，该模型的 mAP@0.5:0.95 略低于旧冠军，因此本文不声称模型在所有指标上全面优于旧模型，而是根据项目应用场景选择更偏召回和视频稳定性的方案。

## 视频时序性验证

针对视频输入，系统加入了时序平滑验证机制。该机制使用相邻帧检测框的 IoU 进行轨迹关联，当目标连续命中达到 stable_hits=3 后才认为形成稳定吸烟相关事件；当目标短时被遮挡或漏检时，允许 bridge_frames=2 的短时补帧，以减少视频结果闪烁。在 109 个 HMDB51 smoke 视频上，hard_finetune 相比旧冠军多命中 2 个视频事件，平滑后命中帧增加 334 帧，稳定轨迹数从 388 增加到 429，说明时序机制能增强视频演示中的连续性。

## 阈值优化

系统通过参数搜索确定默认运行参数：conf=0.12、IoU=0.45，类别阈值为 cigarette=0.12、smoking_person=0.22、smoke=0.28。该策略降低了 cigarette 小目标的候选框门槛，再通过时序稳定规则过滤短时噪声，适合 CPU 版本部署和 Web Demo 演示。该结论基于正样本视频搜索，不能单独证明误检率下降，因此论文中应表述为“提高视频时序命中和连续性”，而不是“提高精准率”。

## 系统贡献

本项目的主要贡献包括：构建了三类吸烟相关目标检测数据流程；完成 baseline、ECA、SE、hard fine-tune 等多组模型比较；实现了图片检测、视频异步任务、检测记录、模型切换和中文视频报告的 Web 管理系统；在视频层面补充了连续命中帧、稳定轨迹、平滑前后检测数量等时序指标，使项目从单帧检测扩展到更贴近实际使用的视频吸烟行为验证。

## 局限性

当前系统仍存在三点限制：第一，香烟目标面积小，低清晰度和遮挡场景仍容易漏检；第二，视频参数搜索主要使用 HMDB51 smoke 正样本，负样本误检验证不足；第三，手工 hard-case 修框方案已取消，因此数据质量提升空间仍然存在。后续工作可优先补充非吸烟视频测试、误检样本分析和少量高质量人工修框数据。
